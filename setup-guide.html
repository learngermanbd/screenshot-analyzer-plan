Supabase — Database, Auth & Storage

Supabase provides a PostgreSQL database, authentication, and file storage all in one free package.

https://supabase.com

Step-by-Step Setup

  1. Go to supabase.com and click "Start your project"
  2. Sign up with your GitHub account (recommended) or email
  3. Once logged in, click "New Project" on the dashboard
  4. Fill in the project details:
    # Project Settings Organization: Select or create an organization Project Name: screenshot-analyzer Database Password: [Create a strong password - SAVE THIS!] Region: Choose closest to your users (e.g., US East, EU West)
  5. Click "Create new project" and wait ~2 minutes for setup
  6. Once ready, go to Settings → API in the left sidebar
  7. Copy these values (you'll need them later):
    # Copy these values Project URL: https://xxxxxxxx.supabase.co Anon Key: eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9... Service Role Key: eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

What You Get (Free Tier)

Database

  • 500 MB PostgreSQL storage
  • Dedicated database instance
  • SQL editor in dashboard
  • Auto-pauses after 1 week inactivity

Auth & Storage

  • 50,000 monthly active users
  • Social logins (Google, GitHub)
  • 1 GB file storage
  • 2 projects max on free tier
Important: Save your Database Password somewhere safe. You cannot retrieve it later. Also save the Service Role Key — it bypasses RLS policies (keep it secret).

Environment Variables to Save

NEXT_PUBLIC_SUPABASE_URL→Your Project URL
NEXT_PUBLIC_SUPABASE_ANON_KEY→Your Anon Key
SUPABASE_SERVICE_ROLE_KEY→Your Service Role Key
DATABASE_URL→postgresql://postgres:[PASSWORD]@db.[REF].supabase.co:5432/postgres

Vercel — Frontend Hosting

Vercel is the best hosting platform for Next.js apps. The Hobby plan is free for personal/non-commercial use.

https://vercel.com

Step-by-Step Setup

  1. Go to vercel.com and click "Sign Up"
  2. Sign up with your GitHub account (recommended)
  3. Once logged in, you'll see your dashboard
  4. Click "Add New..." → "Project"
  5. Import your GitHub repo (e.g., screenshot-analyzer-plan)
  6. Configure the project:
    # Project Settings Framework Preset: Next.js (auto-detected) Root Directory: ./ Build Command: npm run build Output Directory: .next
  7. Add your Environment Variables (from Supabase, Gemini, etc.)
  8. Click "Deploy" — your site will be live in ~1 minute

What You Get (Hobby Plan)

Hosting

  • 100 GB bandwidth/month
  • Automatic HTTPS/SSL
  • Global CDN
  • Custom domain support
  • *.vercel.app subdomain

Serverless Functions

  • 4 hours Active CPU/month
  • 100 GB-hours execution
  • API routes work automatically
  • Edge functions supported
Limitation: Vercel Hobby is for non-commercial use only. If you plan to monetize, you'll need the Pro plan ($20/month). For development and testing, Hobby is perfect.
Tip: Vercel automatically deploys on every push to your main branch. No CI/CD setup needed.

Cloudflare R2 — Image Storage

R2 is S3-compatible object storage with zero egress fees. This is critical for an image-heavy app — other providers charge per GB downloaded.

https://developers.cloudflare.com/r2

Step-by-Step Setup

  1. Go to dash.cloudflare.com and sign up (or log in)
  2. From the dashboard, click "R2" in the left sidebar
  3. Click "Create bucket"
    # Bucket Settings Bucket Name: screenshot-analyzer-images Location: Automatic (or choose nearest region)
  4. After creating the bucket, go to "R2 Overview"
  5. Click "Manage R2 API Tokens"
  6. Click "Create API token"
    # Token Settings Token Name: screenshot-analyzer-token Permissions: Object Read & Write Specify: Only the bucket you created TTL: None (no expiration)
  7. Copy the credentials shown (shown only once!):
    # Copy these values Access Key ID: xxxxxxxxxxxxxxxxxxxxxxxx Secret Access Key: xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx Endpoint: https://<ACCOUNT_ID>.r2.cloudflarestorage.com

What You Get (Free Tier)

  • 10 GB total storage
  • Unlimited Class A operations (writes) — 1M/month free
  • Unlimited Class B operations (reads) — 10M/month free
  • $0 egress fees (unlimited downloads — this is huge!)
  • S3-compatible API (works with AWS SDK)
Why R2 over AWS S3? AWS S3 charges $0.09/GB for egress. If your app serves 100GB of images/month, that's $9/month on S3 but $0 on R2.

Environment Variables

R2_ACCESS_KEY_ID→Your Access Key ID
R2_SECRET_ACCESS_KEY→Your Secret Access Key
R2_ENDPOINT→https://<ACCOUNT_ID>.r2.cloudflarestorage.com
R2_BUCKET_NAME→screenshot-analyzer-images

Upstash Redis — Caching

Upstash provides serverless Redis for caching analysis results. This saves API costs by avoiding re-analysis of the same image.

https://upstash.com

Step-by-Step Setup

  1. Go to upstash.com and click "Sign Up"
  2. Sign up with your GitHub account
  3. From the dashboard, click "Create Database"
    # Database Settings Name: screenshot-analyzer-cache Type: Regional Region: Choose closest (e.g., us-east-1)
  4. After creation, click on the database name
  5. Scroll to "REST API" section and copy:
    # Copy these values UPSTASH_REDIS_REST_URL: https://xx-xxx-xxx.upstash.io UPSTASH_REDIS_REST_TOKEN: AYxxASxxxxxxxxxxxxxxxxxxxxxxxx

What You Get (Free Tier)

  • 500,000 commands per month
  • 256 MB storage
  • Global replication available
  • No credit card required

How Caching Saves Money

# Without cache: every upload = 1 Gemini API call User uploads same image twice = 2 API calls # With Redis cache: hash image, check cache first User uploads same image twice = 1 API call + 1 cache hit ($0) # 500K commands/month = ~16K unique analyses + cache lookups

Environment Variables

UPSTASH_REDIS_REST_URL→Your REST URL
UPSTASH_REDIS_REST_TOKEN→Your REST Token

GitHub — Code Repository

GitHub hosts your source code and integrates with Vercel for automatic deployments.

https://github.com

Step-by-Step Setup

  1. Go to github.com and sign up (or log in)
  2. Click "+" → "New repository"
  3. Configure the repo:
    # Repository Settings Name: screenshot-analyzer Description: Mobile App Screenshot Analyzer & Design Builder Visibility: Public (free) or Private (also free) Initialize: Add a README file
  4. Click "Create repository"

GitHub Personal Access Token (for pushing)

  1. Go to Settings → Developer settings → Personal access tokens → Fine-grained tokens
  2. Click "Generate new token"
  3. Set permissions: Contents: Read and write, Metadata: Read-only
  4. Select your repository
  5. Copy the token (shown only once!)
Security: NEVER share your GitHub token or commit it to a repository. Use environment variables only. If a token is exposed, revoke it immediately at github.com/settings/tokens.

Verification Checklist

Account Verification

  • Supabase project created and URL + keys copied
  • Vercel account created and connected to GitHub
  • Gemini API key generated from Google AI Studio
  • Cloudflare R2 bucket created and API token generated
  • Upstash Redis database created (if using caching)
  • Hugging Face Space created for ML service
  • Sentry project created and DSN copied (if using monitoring)
